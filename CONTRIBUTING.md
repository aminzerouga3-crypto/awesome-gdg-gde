## How Can I Contribute?

### Adding a Resource
If you have a Codelab, Tech Talk, or useful link to share:
1. **Search the list:** Check if the resource is already there.
2. **Fork the repository:** Create your own copy to work on.
3. **Create a branch:** Name it something descriptive like `add-gemini-codelab`.
4. **Update the README:** Add your entry to the appropriate table. Please ensure:
   - The link works.
   - The creator/speaker is properly credited with a link to their profile (GitHub/Twitter/LinkedIn).
   - Use a clear, concise description.
5. **Submit a Pull Request (PR):** Describe what you added and why it's useful for the community.

### Contribution Rules
- **Quality over Quantity:** Only suggest high-quality resources that genuinely help developers.
- **Respect the Branding:** Ensure any links to official GDG assets follow Google's branding guidelines.
- **Be Inclusive:** We follow the [Google Open Source Code of Conduct](https://opensource.google/conduct/).

## Pull Request Checklist
- [ ] My link is relevant to GDG India or Google Technologies.
- [ ] I have added my entry in the correct section.
- [ ] I have included the creator/speaker details.
- [ ] The Markdown is formatted correctly (check the table alignment!).
